Terms Definition
software engineering the branch of computer science that seeks principles to guide the development of large, complex software systems
integrated development environment(IDE) combine tools for developing software into a single, integrated package.

Software Life Cycle: Development phase
Requirement analysis specify what services the proposed system will provide, to identify any conditions
Design creating a plan for the construction of the proposed system
Implementation the actual writing of programs, creation of data files, and development of databases
Testing result of each intermediate step in the entire development process should be “tested” for accuracy
Terms Definition
modular implementation software system is viewed as a collection of units, called objects.
Unifed Modeling Language (UML) developed with the object-oriented paradigm
use case interactions between the system
use case diagram to capture the image of the proposed system from the user’s point of view
data relationships
One-to-one exemplified by the association between patients and occupied private rooms in that each patient is associated with only one room and each room is associated with only one patien
one-to-many by the association between physicians and patients in that one physician is associated with many patients and each patient is associated with onephysician
many-to-many each physician could be associated with many patients and each patient could be associated with many physicians.
Terms Definition
design pattern a predeveloped model for solving a recurring problem in software design.
Software quality assurance (SQA) overseeing and enforcing the quality control systems adopted by the organization
Software testing
glass box testing software tester is aware of the interior structure of the software and uses this knowledge when designing the test
black box testing tests that do not rely on knowledge of the software’s interior composition
beta testing a preliminary version of the software is given to a segment of the intended audience with the goal of learning how the software performs in real-life situations before the final version of the product is solidified and released to the market.
alpha testing similar to beta testing, performed at the developer’s site
Documentation
user documentation the features of the software and describe how to use them.
system documentation describe the software’s internal composition so that the software can be maintained later in its life cycle
technical documentation describe how a software system should be installed and serviced.
Intellectual property law
software license a legal agreement between the owner and user of a software product that grants the user certain permissions to use the product without transferring ownership rights to the intellectual property
software disclaimer denying liability for any damages from using the software
best defenses against software liability claims to apply sound software engineering principles during the software’s development, to use a level of care compatible with the software’s application, and to produce and maintain records that validate these endeavors.
